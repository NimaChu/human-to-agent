# Human to Agent method contract

Methods start from a real case and direct evidence. Treat onboarding states as routing states, skip already-known states, and ask at most one necessary question. When enough evidence exists, act without manufacturing another question. Before activation, discoveries and Unknowns are provisional conversation context. Once a concrete goal and real material exist, announce a safe collision-free slug, run the canonical `hta workspace new` operation, capture supplied text or files durably with `hta capture record`, and materialize confirmed constraints and every provisional Unknown as normative assets. Record the evidence basis and cheapest probe for uncertainty; never invent a rule to make a gate pass. Deviations use plan, discovery, conservative-choice, and revisit fields. Evaluation is independent from implementation. Validate and inspect the diff before `hta record-change`; persistent state belongs in normative source files, not chat history.
